# 簡単な履歴

30歳の時、営業からソフトウェアエンジニアに転職。SESの会社に入社し、WebサービスやEC向けサービス開発に従事してきた。

# 22ヶ月経ってどうか

もう直ぐ2年目が終わろうとしている。ただし自分の中では思うように成長できていないという焦りがある。それをなぜか考えてみた。

一つには、「何ができますか？」と聞かれた時に即答できるような強みがないことだ。例えば、「3年間AWSに携わってきて資格もx個持っています」とか、「xxxのプロジェクト経験3年でoooができます」とか、そういうことを言うことができていない。

理由は明確で、一つには自分が知らないことを知るために色々な知識を身につけようとした結果、**浅く広くの知識**になってしまっていること。もう一つは、SESとして半年単位でさまざまなプロジェクトに関わった結果、**業務で得られる経験も分散**してしまったこと、がある。

別に無駄な知識ではないし、その時その時では業務を行う上で役に立つ知識だった。個人の時間を使って時々で使った技術資格もいくつか取得した。(AWSのソリューションアーキテクトアソシエイトなど)

このまま続けても、会社内で業務を回せるようにはなると思う。ただし、それだけで終わってしまう。SESで業務を回す、と言うのは、エンジニアとしてのレベルアップとは少しずれている気がする。もちろん会社として大切なことなのだが、シニアエンジニアが求められる**「システム全体や大きな機能のデザイン、設計」**と言うのはなかなかSESの立場では得づらい経験だとも思う。

# 何を目指しているのか

自分としては、3年内でジュニアエンジニアを卒業したいと思っている。ジュニア、ミドルの定義はネット上でいろいろあるが、自分としてはジュニアの定義は以下に近いと思っている。

**「周りのサポートを受けながら小さな機能の新規設計、実装ができる。複雑ではない既存の機能改修が一人でできる。」**

浅く広くなってしまった反省を生かして、2026年は**システム全体をデザインする力**、そしてそれを**コードに落とす実装力**の2つに絞って自分の力を伸ばそうと思う。

# 脳裏によぎる美化される過去

時折、営業時代に成績を出して評価されていたことを思い出す。当時と比べると、今は仕事をしてはフィードバックやレビューを受け、自分の出来なさに毎回凹む。なんでこんなにも自分の成長速度が遅いのだろう、と悩むことは多い。

そんな時にどうしても、あのまま営業を続けていたら〜、と思うことはある。当時は当時で絶対に辛いことなどはあったはずなのだが、人間どうしたものか、過去の辛いことは忘れて楽しかったことは簡単に思い出せるようになっているらしい。都合の良いものである。

# じゃあどうするか

泣き言を言っていても始まらないので、残り1年とちょっと本気で頑張ってミドルレベルのエンジニアを目指そうと思う。もしそれでダメだったら潔く、営業に戻ることを検討する。

中途半端に挑戦してダメでした、はキャリア的にマイナスでしかないし、多分将来的にも「あの時本気出していたらなぁ」と居酒屋でクダをまくおじさんになってしまう危険性があるため、本気で挑戦してダメだったらその時は**戦略的撤退**をする。

ちょうどプライベートも節目を迎えそうなので、最後の挑戦の年としてはちょうどキリが良いのかもしれない。その時は、家族が一人増えているはずなので、相談して次のキャリアを考える予定である。
